# Multi-Megastructures + Free Tech — Stellaris Mod

A Stellaris mod for **Cetus v4.3.x** with three effects:

1. **Multiple of each megastructure.** Lifts the one-per-empire cap so a single empire can build, e.g., two Dyson Spheres.
2. **Multiple megastructures per system / celestial body.** Removes the per-system uniqueness gate so a system can host more than one multi-stage megastructure.
3. **All empires start with megastructure tech.** Every player and AI empire is granted every megastructure-unlocking technology (including Mega-Engineering and all Galactic-Wonders-gated techs) on game start.

## How it works

Two pieces:

- `mod/common/megastructures/*.txt` — overrides for the 14 vanilla megastructure files that contain build-limit gates. The overrides are generated by `scripts/build.py` from the locally installed vanilla files. The script rewrites every `built_<name>` country-flag reference to an inert sentinel flag (`mmegs_unused_flag`) we never set, which makes the existing `NOT/NOR` gates always pass and the flag-setting effects no-op. The per-system gate (`has_no_non_gate_megastructure = yes`) is rewritten to `always = yes`.
- `mod/common/on_actions/zz_mmegs_on_actions.txt` + `mod/events/mmegs_events.txt` — hook `on_game_start_country` to fire a hidden event that calls `give_technology` for every megastructure tech on every regular empire.

This keeps the mod patch-resilient: when Stellaris updates, re-run `scripts/build.py` against the new vanilla files and the same transformations apply.

## Build

Generates the megastructure overrides from your local Stellaris install:

```bash
python3 scripts/build.py
```

Reads from `~/Library/Application Support/Steam/steamapps/common/Stellaris/common/megastructures/`. Edit the `VANILLA_DIR` constant in `scripts/build.py` if your install is elsewhere.

## Pre-game verification

```bash
python3 scripts/preflight.py
```

Ten automated checks — brace balance, leaked limit markers, vanilla key drift, tech-ID existence, on_action existence, event refs, localisation BOM, descriptor sanity, and (if deployed) symlink integrity. Full table in [`TESTING.md`](TESTING.md).

## Deploy locally

Symlinks the mod into your Stellaris user directory and writes the outer launcher descriptor:

```bash
scripts/deploy.sh
```

This creates:

- `~/Documents/Paradox Interactive/Stellaris/mod/multi-megastructures` → symlink to `apps/mods/stellaris/multi-megastructures/mod`
- `~/Documents/Paradox Interactive/Stellaris/mod/multi-megastructures.mod` → launcher descriptor with absolute `path=`

Re-running `scripts/build.py` is picked up immediately by the launcher — no need to re-deploy.

## Test in-game

1. Launch Stellaris → Paradox Launcher → Mods → enable **Multi-Megastructures + Free Tech** → Play.
2. Start a new game.
3. **Feature 3 check:** open Research → all megastructure techs (Mega-Engineering, Dyson Sphere, Ring World, Matter Decompressor, Think Tank, Strategic Coordination, Mega Art, Interstellar Assembly, Mega Shipyard, Quantum Catapult, Orbital Arc Furnace, Dyson Swarm, Galactic Archivism, Deep Space Citadel, Cosmogenesis World) are already researched. Use the `play <country_id>` console command to confirm AI empires have them too.
4. **Feature 1 check:** build a Dyson Sphere. After completion, start a second Dyson Sphere in a different system — the construction option must still be available.
5. **Feature 2 check:** in a system that has a finished megastructure, try to build a second (different) megastructure on a different celestial body in the same system. Then try on the same celestial body. Both should work.
6. Tail `~/Documents/Paradox Interactive/Stellaris/logs/error.log` and check for unresolved trigger errors after starting the game.

## Known caveats

- **AI behavior.** Removing the cap doesn't necessarily make the AI build extras. Vanilla AI weights (`ai_weight` blocks in each megastructure file) were tuned for the one-per-empire world. The AI may stop at one even though the gate is gone. If you want AI empires to spam megastructures, tweaking those weights is a follow-on.
- **Mod conflicts.** Any other mod that overrides `common/megastructures/*.txt` will conflict — last-loaded wins.
- **Gateways, orbital rings, hyper relays, habitats.** These were already multi-build by design in vanilla; they're not part of the override set.
